I. NGƯỜI DÙNG & XÁC THỰC
================================================================================

[T01] Bảng: users
--------------------------------------------------------------------------------
Lưu thông tin tài khoản của tất cả người dùng (cả bệnh nhân lẫn nhân viên). Nhân viên sẽ có thêm row
tương ứng trong bảng staffs. Trường user_type giúp xác định nhanh loại người dùng mà không cần JOIN thêm.
API: Login, Signup, Get/Set Profile, Delete Account.

Trường            | Kiểu dữ liệu            | Ràng buộc                          | Mô tả
------------------|-------------------------|-------------------------------------|----------------------------------------------
user_id           | BIGINT UNSIGNED         | PK, AUTO_INCREMENT                  | Khóa chính người dùng
phone_number      | VARCHAR(15)             | UNIQUE NOT NULL                     | Số điện thoại đăng ký, duy nhất, dùng để đăng nhập
password_hash     | VARCHAR(255)            | NOT NULL                            | BCrypt hash mật khẩu (cost=12)
full_name         | VARCHAR(100)            | NOT NULL                            | Họ tên đầy đủ
user_type         | ENUM('patient','staff') | NOT NULL DEFAULT patient            | Phân loại nhanh: patient = bệnh nhân, staff = nhân viên
date_of_birth     | DATE                    | NULL                                | Ngày sinh (hồ sơ bệnh án)
gender            | ENUM('M','F','O')       | NULL                                | Giới tính: M/F/O
avatar_url        | VARCHAR(500)            | NULL                                | URL ảnh đại diện
status            | ENUM(...)               | NOT NULL DEFAULT active             | Trạng thái tài khoản. pending = chờ xác thực OTP
created_at        | DATETIME                | NOT NULL                            | Thời điểm đăng ký
updated_at        | DATETIME                | NULL                                | Lần cập nhật gần nhất


[T02] Bảng: otp_codes
--------------------------------------------------------------------------------
Lưu mã OTP xác thực. Trong MVP server trả otp_code plain text qua otp_debug trong response thay vì gửi
SMS thật. Bảng này vẫn cần thiết để quản lý trạng thái, tránh sử dụng lại mã đã dùng (replay attack).
API: SMS Verify, Forgot Password.

Trường        | Kiểu dữ liệu                               | Ràng buộc          | Mô tả
--------------|---------------------------------------------|--------------------|-------------------------------
otp_id        | BIGINT UNSIGNED                             | PK, AUTO_INCREMENT | Khóa chính
phone_number  | VARCHAR(15)                                 | NOT NULL INDEX     | Số điện thoại nhận OTP
otp_code      | VARCHAR(8)                                  | NOT NULL           | Mã OTP lưu dạng hash 1 chiều
type          | ENUM('signup','reset_password','change_phone') | NOT NULL        | Loại OTP, tránh dùng nhầm mã signup cho reset_password
expired_at    | DATETIME                                    | NOT NULL           | Thời điểm hết hạn (10 phút trong MVP)
is_used       | TINYINT(1)                                  | NOT NULL DEFAULT 0 | Đã dùng chưa: 0 = chưa, 1 = đã dùng
created_at    | DATETIME                                    | NOT NULL           | Thời điểm tạo


[T03] Bảng: user_settings
--------------------------------------------------------------------------------
Lưu cấu hình cá nhân của người dùng. Quan hệ one-to-one với users. Row này tự động tạo khi user đăng ký
thành công với tất cả giá trị default. travel_mode ảnh hưởng trực tiếp đến thuật toán tìm đường.
API: Get/Set App Settings.

Trường                   | Kiểu dữ liệu                       | Ràng buộc                  | Mô tả
-------------------------|------------------------------------|-----------------------------|---------------------------------
setting_id               | BIGINT UNSIGNED                    | PK, AUTO_INCREMENT          | Khóa chính
user_id                  | BIGINT UNSIGNED                    | FK users(user_id) UNIQUE    | 1 user chỉ có 1 dòng setting
voice_guidance_enabled   | TINYINT(1)                         | NOT NULL DEFAULT 1          | Bật/tắt chỉ đường giọng nói
notification_enabled     | TINYINT(1)                         | NOT NULL DEFAULT 1          | Bật/tắt thông báo đẩy
travel_mode              | ENUM('walk','wheelchair','stretcher') | NOT NULL DEFAULT walk    | Chế độ di chuyển mặc định, ảnh hưởng thuật toán tìm đường
language                 | VARCHAR(10)                        | NOT NULL DEFAULT vi         | Ngôn ngữ giao diện: vi / en
updated_at               | DATETIME                           | NULL                        | Cập nhật lần cuối


[T04] Bảng: fcm_tokens
--------------------------------------------------------------------------------
Lưu Firebase Cloud Messaging token của từng thiết bị di động. Một user có thể có nhiều row (nhiều thiết bị).
Trong MVP chưa gửi push notification nhưng vẫn lưu để giám sát loại thiết bị và sẵn sàng tích hợp FCM sau này.
API: Set Device Token, Logout.

Trường           | Kiểu dữ liệu           | Ràng buộc              | Mô tả
-----------------|------------------------|-------------------------|------------------------------------

user_id          | BIGINT UNSIGNED        | FK users(user_id) NOT NULL | Chủ token
fcm_token        | VARCHAR(500)           | NOT NULL UNIQUE         | Token FCM từ Firebase SDK
device_platform  | ENUM('ios','android')  | NOT NULL               | Nền tảng thiết bị
device_model     | VARCHAR(100)           | NULL                    | Model máy (Samsung S24, iPhone 15...)
app_version      | VARCHAR(20)            | NULL                    | Phiên bản app đang cài
is_active        | TINYINT(1)             | NOT NULL DEFAULT 1      | Token còn hiệu lực. Set 0 khi logout
created_at       | DATETIME               | NOT NULL               | Ngày đăng ký token
last_used_at     | DATETIME               | NULL                    | Lần dùng gần nhất

Trường       | Kiểu dữ liệu                        | Ràng buộc                     | Mô tả

role         | ENUM('admin','coordinator','staff') | NOT NULL                       | admin: toàn quyền | coordinator: giám sát | staff: hỗ trợ

created_at   | DATETIME                            | NOT NULL                       | Ngày tạo tài khoản nhân viên


[T06] Bảng: wards
--------------------------------------------------------------------------------
Danh mục khoa/viện của bệnh viện. Dùng để nhóm các phòng khám theo chuyên khoa và phân công nhân viên.
Lưu ý circular dependency: staffs.ward_id -> wards và wards.head_staff_id -> staffs.
Khi seed data phải INSERT ward với head_staff_id=NULL trước, tạo staff xong rồi UPDATE.

Trường         | Kiểu dữ liệu    | Ràng buộc                       | Mô tả
---------------|-----------------|----------------------------------|---------------------------------
ward_id        | INT UNSIGNED    | PK, AUTO_INCREMENT               | Khóa chính
ward_code      | VARCHAR(20)     | UNIQUE NOT NULL                  | Mã khoa (XN, PTTT, NI...)
ward_name      | VARCHAR(200)    | NOT NULL                         | Tên đầy đủ (Khoa Nội tổng hợp...)
head_staff_id  | BIGINT UNSIGNED | FK staffs(staff_id) NULL         | Trưởng khoa. NULL khi mới tạo
is_active      | TINYINT(1)      | NOT NULL DEFAULT 1               | Đang hoạt động

[T09] Bảng: floors
--------------------------------------------------------------------------------
Tầng của từng tòa nhà. Mỗi tầng là 1 layer bản đồ riêng biệt. Cung cấp ảnh nền và hệ số quy đổi pixel -> mét.

Công thức quy đổi pixel -> mét:
  meter_x = pixel_x × (real_width_m / image_width_px)
  meter_y = pixel_y × (real_height_m / image_height_px)

Trường           | Kiểu dữ liệu  | Ràng buộc         | Mô tả
-----------------|---------------|--------------------|-------------------------------------------------
floor_id         | INT UNSIGNED  | PK, AUTO_INCREMENT | Khóa chính
building_id      | INT UNSIGNED  | FK buildings(building_id) | Tòa nhà chứa tầng này
floor_number     | TINYINT       | NOT NULL           | Số tầng (0 = trệt, 1 = lầu 1...)
floor_name       | VARCHAR(100)  | NOT NULL           | Tên tầng hiển thị trên App (VD: "Tầng 1")
display_order    | INT           | NOT NULL DEFAULT 0 | Số thứ tự ưu tiên sắp xếp bộ lọc tầng trên App
map_image_url    | VARCHAR(500)  | NULL               | URL ảnh nền sơ đồ tầng (.png/.jpg/.svg)
image_width_px   | INT           | NOT NULL           | Chiều rộng ảnh nền theo pixel
image_height_px  | INT           | NOT NULL           | Chiều cao ảnh nền theo pixel
real_width_m     | FLOAT         | NOT NULL DEFAULT 0 | Chiều rộng thực tế của tầng theo mét
real_height_m    | FLOAT         | NOT NULL DEFAULT 0 | Chiều cao thực tế của tầng theo mét
is_active        | TINYINT(1)    | NOT NULL DEFAULT 1 | Đang sử dụng

[T11] Bảng: map_edges
--------------------------------------------------------------------------------
Hành lang/đoạn đường nối 2 node. Có thể cùng 1 tầng hoặc cross-floor.
weight = trọng số dùng trong thuật toán Dijkstra.

Công thức tính distance_m:
  dx = (access_x_to - access_x_from) × (real_width_m / image_width_px)
  dy = (access_y_to - access_y_from) × (real_height_m / image_height_px)
  distance_m = sqrt(dx² + dy²)

Trường                | Kiểu dữ liệu  | Ràng buộc                       | Mô tả
----------------------|---------------|---------------------------------|-------------------------------------------------
edge_id               | INT UNSIGNED  | PK, AUTO_INCREMENT              | Khóa chính
floor_id              | INT UNSIGNED  | FK floors(floor_id) INDEX       | Tầng quản lý. Nếu cross_floor thì lưu tầng xuất phát
from_node_id          | INT UNSIGNED  | FK map_nodes(node_id)           | Node bắt đầu
to_node_id            | INT UNSIGNED  | FK map_nodes(node_id)           | Node kết thúc
polygon_coords        | TEXT          | NULL                            | Tọa độ polygon mô tả vùng hành lang (tuỳ chọn)
distance_m            | FLOAT         | NOT NULL DEFAULT 0              | Khoảng cách thực tế giữa 2 node theo mét
weight                | FLOAT         | NOT NULL DEFAULT 1.0            | Trọng số cho thuật toán Dijkstra
is_bidirectional      | TINYINT(1)    | NOT NULL DEFAULT 1              | Đường hai chiều (1) hay một chiều (0)
