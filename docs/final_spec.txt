================================================================================
PHẦN A — DANH SÁCH API
================================================================================

Quy ước:
  Method : POST / GET / DELETE / PATCH
  Auth   : X = Public (không cần token)
           O = Private (cần Bearer JWT token)
           A = Admin (cần JWT + role admin/coordinator)

Format response chuẩn:
  { "code": 1000, "message": "OK", "data": { ... } }

NHÓM 1 — XÁC THỰC & TÀI KHOẢN (15 API)
────────────────────────────────────────────────────────────────────────────────

  1. POST X api/auth/login
     → READ users WHERE phone_number, so sánh password_hash
     → Trả JWT token chứa user_id

  2. POST O api/auth/logout
     → UPDATE fcm_tokens SET is_active=0 WHERE user_id + fcm_token
     → Hủy JWT (cần Redis blacklist hoặc short-lived token)

  3. POST X api/auth/signup
     → INSERT users (status=pending)
     → INSERT otp_codes (type=signup, expired_at=now+10min)
     → Trả otp_debug trong response (MVP không gửi SMS)

  4. POST X api/auth/verify_otp
     → READ otp_codes WHERE phone_number + otp_code + is_used=0
     → UPDATE users SET status=active
     → UPDATE otp_codes SET is_used=1
     → INSERT user_settings (default values)
     → Trả JWT token

  5. POST X api/auth/resend_otp
     → UPDATE otp_codes SET is_used=1 WHERE phone_number (vô hiệu cũ)
     → INSERT otp_codes mới

  6. POST X api/auth/forgot_password
     → READ users WHERE phone_number (kiểm tra tồn tại)
     → INSERT otp_codes (type=reset_password)

  7. POST X api/auth/reset_password
     → READ otp_codes WHERE phone_number + type=reset_password
     → UPDATE users SET password_hash
     → UPDATE otp_codes SET is_used=1

  8. POST O api/auth/change_password
     → READ users WHERE user_id, so sánh password_hash cũ
     → UPDATE users SET password_hash mới

  9. GET O api/user/get_profile
     → READ users WHERE user_id
     → LEFT JOIN staffs (nếu user_type=staff)

 10. POST O api/user/set_profile
     → UPDATE users SET full_name, avatar_url, date_of_birth, gender

 11. POST O api/user/set_devtoken
     → UPSERT fcm_tokens (INSERT hoặc UPDATE nếu token đã tồn tại)

 12. GET O api/user/get_settings
     → READ user_settings WHERE user_id

 13. POST O api/user/set_settings
     → UPDATE user_settings SET voice_guidance, notification, travel_mode, language

 14. GET X api/sys/check_version
     → READ app_versions WHERE platform ORDER BY version_code DESC LIMIT 1
     → So sánh version_code client gửi lên

 15. DELETE O api/user/delete_account
     → UPDATE users SET status=blocked (soft delete)
     → UPDATE fcm_tokens SET is_active=0
     → Không xóa cứng — giữ data cho audit

NHÓM 2 — BẢN ĐỒ & HẠ TẦNG (15 API)

     → READ map_edges WHERE floor_id + is_active=1

     → Trả: edge_id, from_node_id, to_node_id, distance_m, weight

 19. GET X api/map/get_meta
     → READ floors WHERE floor_id
     → Trả: image_width_px, image_height_px, real_width_m, real_height_m, map_image_url

 20. GET X api/map/get_depts
     → READ wards WHERE is_active=1
     → JOIN map_nodes WHERE ward_id (đếm số phòng mỗi khoa)

 21. GET X api/map/search_location
     → FULLTEXT SEARCH map_nodes WHERE MATCH(node_name) AGAINST(keyword)
     → JOIN floors (floor_name cho kết quả)

 22. GET X api/map/get_landmarks
     → READ map_nodes WHERE is_landmark=1 + is_active=1

 23. GET O api/map/get_beacons
     → Bảng riêng (chưa có trong schema hiện tại, trả data tĩnh)

 24. GET X api/map/sync_full
     → READ floors + map_nodes + map_edges (toàn bộ)
     → Gộp thành 1 JSON lớn để frontend lưu offline

 25. POST A api/admin/add_node
     → INSERT map_nodes

 26. POST A api/admin/edit_node
     → UPDATE map_nodes WHERE node_id

 27. DELETE A api/admin/del_node
     → UPDATE map_nodes SET is_active=0 (soft delete)
     → UPDATE map_edges SET is_active=0 WHERE from/to_node_id

 28. POST A api/admin/add_edge
     → INSERT map_edges
     → Tính distance_m từ tọa độ 2 node + tỷ lệ pixel/m

 29. DELETE A api/admin/del_edge
     → UPDATE map_edges SET is_active=0

 30. PATCH A api/admin/set_weight
     → UPDATE map_edges SET weight WHERE edge_id

NHÓM 3 — ĐIỀU PHỐI LỘ TRÌNH (15 API)
────────────────────────────────────────────────────────────────────────────────

 31. POST O api/route/ordered
     → READ map_nodes + map_edges (build Dijkstra graph)
     → READ travel_modes WHERE mode_id (speed_factor)
     → Chạy Dijkstra từng cặp node theo thứ tự
     → INSERT routes (status=active)
     → INSERT route_paths (bulk, mỗi step gồm instruction + voice_text)
     → Trả: route_id, path_coords, total_distance, estimated_time

 32. POST O api/route/unordered
     → Giống #31 nhưng thêm bước tối ưu thứ tự (TSP/greedy)
     → READ edge_traffic_status (trọng số động theo mật độ)
     → INSERT routes + route_paths

 33. POST O api/route/recalculate
     → READ routes WHERE route_id + user_id
     → DELETE route_paths WHERE route_id (xóa path cũ)
     → Chạy Dijkstra mới với avoid_edges từ request
     → INSERT route_paths mới
     → UPDATE routes SET total_distance, estimated_time

 34. GET O api/route/get_active
     → READ routes WHERE user_id + status=active
     → READ route_paths WHERE route_id

 35. POST O api/route/cancel
     → UPDATE routes SET status=cancelled WHERE route_id + user_id

 36. GET O api/route/get_steps
     → READ route_paths WHERE route_id ORDER BY step_order
     → JOIN map_nodes (node_name cho instruction)
     → JOIN floors (floor_name cho chuyển tầng)
     → Trả: step_order, instruction, voice_text, x, y, floor_name

 37. POST O api/route/preview
     → Giống #31 nhưng KHÔNG INSERT vào DB
     → Chỉ chạy Dijkstra và trả path_coords

 38. POST O api/route/get_eta
     → READ routes WHERE route_id (total_distance, mode_id)
     → READ travel_modes (speed_factor)
     → READ route_history_nodes (tìm current node)
     → Tính: distance_còn_lại ÷ (1.2 × speed_factor)

 39. GET O api/route/get_history
     → READ routes WHERE user_id ORDER BY created_at DESC
     → JOIN map_nodes (start_node_name, dest_node_name)
     → Pagination: page, limit

 40. DELETE O api/route/clear_history
     → SOFT DELETE routes WHERE user_id + status IN (completed, cancelled)

 41. POST O api/route/share
     → READ routes WHERE route_id + user_id
     → INSERT route_shares (share_url=UUID, expired_at=now+24h)

 42. POST O api/route/rate_path
     → INSERT route_feedbacks (route_id UNIQUE — mỗi route chỉ rate 1 lần)

 43. POST O api/route/pass_node
     → INSERT route_history_nodes (status=arrived, arrival_time=now)
     → UPDATE route_history_nodes SET status='passed' cho node trước

 44. GET O api/route/get_next
     → READ routes WHERE user_id + status=active
     → READ route_paths WHERE route_id + step_order > current
     → Trả: next node_name, instruction

 45. GET X api/route/get_modes
     → READ travel_modes (toàn bộ)

 60. GET A api/flow/simulate/status
     → Đọc từ RAM: số goroutine đang chạy, thời gian chạy
     → Không truy vấn DB

I. NGƯỜI DÙNG & XÁC THỰC
────────────────────────────────────────────────────────────────────────────────

[T01] users
  user_id           BIGINT UNSIGNED  PK AUTO_INCREMENT
  phone_number      VARCHAR(15)      UNIQUE NOT NULL
  password_hash     VARCHAR(255)     NOT NULL
  full_name         VARCHAR(100)     NOT NULL
  user_type         ENUM('patient','staff')  NOT NULL DEFAULT 'patient'
  date_of_birth     DATE             NULL
  gender            ENUM('M','F','O')  NULL
  avatar_url        VARCHAR(500)     NULL
  status            ENUM('active','pending','blocked')  NOT NULL DEFAULT 'active'
  created_at        DATETIME         NOT NULL
  updated_at        DATETIME         NULL

  Dùng bởi: #1 #2 #3 #4 #7 #8 #9 #10 #15

[T02] otp_codes
  otp_id            BIGINT UNSIGNED  PK AUTO_INCREMENT
  phone_number      VARCHAR(15)      NOT NULL INDEX
  otp_code          VARCHAR(8)       NOT NULL
  type              ENUM('signup','reset_password','change_phone')  NOT NULL
  expired_at        DATETIME         NOT NULL
  is_used           TINYINT(1)       NOT NULL DEFAULT 0
  created_at        DATETIME         NOT NULL

  Dùng bởi: #3 #4 #5 #6 #7

[T03] user_settings
  setting_id        BIGINT UNSIGNED  PK AUTO_INCREMENT
  user_id           BIGINT UNSIGNED  FK users UNIQUE
  voice_guidance_enabled  TINYINT(1)  NOT NULL DEFAULT 1
  notification_enabled    TINYINT(1)  NOT NULL DEFAULT 1
  travel_mode       ENUM('walk','wheelchair','stretcher')  NOT NULL DEFAULT 'walk'
  language          VARCHAR(10)      NOT NULL DEFAULT 'vi'
  updated_at        DATETIME         NULL

  Dùng bởi: #4 (tạo) #12 #13

[T04] fcm_tokens
  token_id          BIGINT UNSIGNED  PK AUTO_INCREMENT
  user_id           BIGINT UNSIGNED  FK users NOT NULL
  fcm_token         VARCHAR(500)     UNIQUE NOT NULL
  device_platform   ENUM('ios','android')  NOT NULL
  device_model      VARCHAR(100)     NULL
  app_version       VARCHAR(20)      NULL
  is_active         TINYINT(1)       NOT NULL DEFAULT 1
  created_at        DATETIME         NOT NULL
  last_used_at      DATETIME         NULL

  Dùng bởi: #2 #11 #15 + FCM push internal (#71 #73 #83)
