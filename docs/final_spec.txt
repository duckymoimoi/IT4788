TRẠNG THÁI TRIỂN KHAI:
  ✅ Nhóm 1 — Xác thực & Tài khoản (15 API)  → ĐÃ CÓ TRONG CODE
  ✅ Nhóm 2 — Bản đồ & POI (15 API)           → ĐÃ REFACTOR SANG GRID 2D
     (test_map: 24/24 PASS, test_all: 64/64 PASS)
  ☐  Nhóm 3 — Tìm đường (14 API)              → Chưa làm (Slice 2-4)
  ☐  Nhóm 4 — Luồng & Mật độ (15 API)         → Chưa làm (Slice 5)
  ☐  Nhóm 5 — Y tế (10 API)                   → Chưa làm (Slice 6)
  ☐  Nhóm 6 — Thiết bị (12 API)               → Chưa làm (Slice 7)
  ☐  Nhóm 7 — Tiện ích (6 API)                → Chưa làm (Slice 10)
  ☐  Nhóm 8 — Hỗ trợ & Liên lạc (15+WS API)  → Chưa làm (Slice 8-9)
  ☐  Nhóm 9 — Admin Engine (6 API)            → Chưa làm (Slice 11)
================================================================================

Quy ước:
  Method : POST / GET / DELETE / PATCH
  Auth   : X = Public (không cần token)
           O = Private (cần Bearer JWT token)
           A = Admin (cần JWT + role admin/coordinator)

Format response chuẩn:
  { "code": 1000, "message": "OK", "data": { ... } }

Kiến trúc bản đồ & tìm đường:
  • Bản đồ: Grid 2D (file .map format MovingAI), chỉ 1 tầng
  • Dijkstra Mode — tìm đường on-demand trên grid (từ POI → POI)
  • MAPF Mode    — gán patient vào agent pre-computed từ output.json

NHÓM 1 — XÁC THỰC & TÀI KHOẢN (15 API)
────────────────────────────────────────────────────────────────────────────────

  1. POST X api/auth/login
     → READ users WHERE phone_number, so sánh password_hash
     → Trả JWT token chứa user_id

  2. POST O api/auth/logout
     → UPDATE fcm_tokens SET is_active=0 WHERE user_id + fcm_token
     → Hủy JWT (cần Redis blacklist hoặc short-lived token)

  3. POST X api/auth/signup
     → INSERT users (status=pending)
     → INSERT otp_codes (type=signup, expired_at=now+10min)
     → Trả otp_debug trong response (MVP không gửi SMS)

  4. POST X api/auth/verify_otp
     → READ otp_codes WHERE phone_number + otp_code + is_used=0
     → UPDATE users SET status=active
     → UPDATE otp_codes SET is_used=1
     → INSERT user_settings (default values)
     → Trả JWT token

  5. POST X api/auth/resend_otp
     → UPDATE otp_codes SET is_used=1 WHERE phone_number (vô hiệu cũ)
     → INSERT otp_codes mới

  6. POST X api/auth/forgot_password
     → READ users WHERE phone_number (kiểm tra tồn tại)
     → INSERT otp_codes (type=reset_password)

  7. POST X api/auth/reset_password
     → READ otp_codes WHERE phone_number + type=reset_password
     → UPDATE users SET password_hash
     → UPDATE otp_codes SET is_used=1

  8. POST O api/auth/change_password
     → READ users WHERE user_id, so sánh password_hash cũ
     → UPDATE users SET password_hash mới

  9. GET O api/user/get_profile
     → READ users WHERE user_id
     → LEFT JOIN staffs (nếu user_type=staff)

 10. POST O api/user/set_profile
     → UPDATE users SET full_name, avatar_url, date_of_birth, gender

 11. POST O api/user/set_devtoken
     → UPSERT fcm_tokens (INSERT hoặc UPDATE nếu token đã tồn tại)

 12. GET O api/user/get_settings
     → READ user_settings WHERE user_id

 13. POST O api/user/set_settings
     → UPDATE user_settings SET voice_guidance, notification, travel_mode, language

 14. GET X api/sys/check_version
     → READ app_versions WHERE platform ORDER BY version_code DESC LIMIT 1
     → So sánh version_code client gửi lên

 15. DELETE O api/user/delete_account

     → UPDATE fcm_tokens SET is_active=0
     → Không xóa cứng — giữ data cho audit

NHÓM 2 — BẢN ĐỒ & HẠ TẦNG (15 API)
────────────────────────────────────────────────────────────────────────────────

  Bản đồ sử dụng grid 2D (format .map MovingAI).
  Mỗi ô trên grid: "." = đi được, "@" = tường/vật cản.
  POI (Point of Interest) = gán tên/loại cho ô cụ thể trên grid.
  Hệ thống chỉ hỗ trợ 1 tầng (single floor).

 16. GET X api/map/get_floors
     → READ grid_maps WHERE is_active=1
     → Trả: map_id, map_name, rows, cols

 17. GET X api/map/get_nodes
     → READ grid_pois WHERE map_id + is_active=1
     → LEFT JOIN wards (ward_name)
     → Trả: poi_id, poi_name, poi_type, grid_row, grid_col, grid_location

 18. GET X api/map/get_edges
     → Tính adjacency từ grid: mỗi ô "." kề 4 hướng (N,S,E,W)
     → Trả danh sách cặp {from_loc, to_loc, weight=1.0}

 19. GET X api/map/get_meta
     → READ grid_maps WHERE map_id
     → Trả: rows, cols, map_file_path, map_image_url

 20. GET X api/map/get_depts
     → READ wards WHERE is_active=1
     → JOIN grid_pois WHERE ward_id (đếm số POI mỗi khoa)

 21. GET X api/map/search_location
     → FULLTEXT SEARCH grid_pois WHERE MATCH(poi_name) AGAINST(keyword)

 22. GET X api/map/get_landmarks
     → READ grid_pois WHERE is_landmark=1 + is_active=1

 23. GET O api/map/get_beacons
     → Trả data tĩnh (chưa dùng beacon thật)

 24. GET X api/map/sync_full
     → READ grid_maps + grid_pois (toàn bộ)
     → Kèm grid data (mảng 2D: 0=free, 1=obstacle)
     → Gộp thành 1 JSON lớn để frontend lưu offline

 25. POST A api/admin/add_node
     → INSERT grid_pois (gán tên/loại cho 1 ô trên grid)
     → Kiểm tra grid_location phải là ô free (không phải tường)

 26. POST A api/admin/edit_node
     → UPDATE grid_pois WHERE poi_id

 27. DELETE A api/admin/del_node
     → UPDATE grid_pois SET is_active=0 (soft delete)

 28. POST A api/admin/add_edge
     → Edges tự tính từ grid adjacency, không cần thêm thủ công
     → Trả code 2003 "Edges auto-computed from grid"

 29. DELETE A api/admin/del_edge
     → Tương tự #28
     → Trả code 2003

 30. PATCH A api/admin/set_weight
     → UPDATE grid_pois SET custom_weight (penalty cho khu vực)
     → Ảnh hưởng Dijkstra: cells xung quanh POI có weight cao hơn

NHÓM 3 — ĐIỀU PHỐI LỘ TRÌNH (15 API)
────────────────────────────────────────────────────────────────────────────────

  Hai chế độ tìm đường:
  • Dijkstra — chạy on-demand trên grid, nhanh, cho single user
  • MAPF     — gán patient vào agent pre-computed, cho simulation

  API route/ordered tự chọn mode:
  • Nếu simulation đang chạy → MAPF mode (gán agent)
  • Nếu không → Dijkstra mode (tính đường mới)

 31. POST O api/route/ordered
     [DIJKSTRA MODE]
     → READ grid_maps + grid_pois (build grid graph in memory)
     → READ travel_modes WHERE mode_id (speed_factor)
     → Chạy Dijkstra trên grid: start_loc → dest_loc
     → Convert path (list of grid_locations) → list of [row, col]
     → INSERT routes (status=active)
     → INSERT route_paths (mỗi step = 1 grid cell + instruction)
     → Trả: route_id, path_coords, total_distance, estimated_time

     [MAPF MODE]
     → AgentManager.AssignPatient(user_id)
     → Lấy trajectory pre-computed từ output.json
     → INSERT routes (status=active, route_mode=mapf)
     → INSERT route_paths từ trajectory
     → Trả: route_id, path_coords, agent_id

 32. POST O api/route/unordered
     → Giống #31 Dijkstra mode
     → Nhiều điểm đích → chạy Dijkstra từng cặp, chọn thứ tự tối ưu

 33. POST O api/route/recalculate
     → READ routes WHERE route_id + user_id
     → DELETE route_paths WHERE route_id (xóa path cũ)
     → Chạy Dijkstra mới (chỉ Dijkstra mode)
     → INSERT route_paths mới
     → UPDATE routes SET total_distance, estimated_time

 34. GET O api/route/get_active
     → READ routes WHERE user_id + status=active
     → READ route_paths WHERE route_id
     → Nếu route_mode=mapf: kèm current_timestep

 35. POST O api/route/cancel
     → UPDATE routes SET status=cancelled WHERE route_id + user_id
     → Nếu route_mode=mapf: AgentManager.ReleasePatient(user_id)

 36. GET O api/route/get_steps
     → READ route_paths WHERE route_id ORDER BY step_order
     → Trả: step_order, grid_row, grid_col, instruction

 37. POST O api/route/preview
     → Giống #31 nhưng KHÔNG INSERT vào DB
     → Chỉ chạy Dijkstra và trả path_coords

 38. POST O api/route/get_eta
     → READ routes WHERE route_id
     → Dijkstra: remaining_steps × avg_step_time ÷ speed_factor
     → MAPF: remaining_steps × tickRate

 39. GET O api/route/get_history
     → READ routes WHERE user_id ORDER BY created_at DESC
     → JOIN grid_pois (start_poi_name, dest_poi_name)
     → Pagination: page, limit

 40. DELETE O api/route/clear_history
     → SOFT DELETE routes WHERE user_id + status IN (completed, cancelled)

 41. POST O api/route/share
     → READ routes WHERE route_id + user_id
     → INSERT route_shares (share_url=UUID, expired_at=now+24h)

 42. POST O api/route/rate_path
     → INSERT route_feedbacks (route_id UNIQUE — mỗi route chỉ rate 1 lần)

 43. POST O api/route/pass_node
     → INSERT route_history_nodes (status=arrived, arrival_time=now)
     → UPDATE route_history_nodes SET status='passed' cho node trước

 44. GET O api/route/get_next
     → READ routes WHERE user_id + status=active
     → READ route_paths WHERE route_id + step_order > current
     → Trả: next poi_name, instruction

 45. GET X api/route/get_modes
     → READ travel_modes (toàn bộ)

IV. BẢN ĐỒ SỐ — GRID 2D
────────────────────────────────────────────────────────────────────────────────

  Bản đồ là grid 2D (file .map format MovingAI).
  Chỉ hỗ trợ 1 tầng (single floor).
  POI = admin gán tên/loại cho ô cụ thể.
  Edges tự tính từ grid adjacency (4 hướng N/S/E/W).

[T08] grid_maps
  map_id            INT UNSIGNED     PK AUTO_INCREMENT
  map_name          VARCHAR(200)     NOT NULL
  map_file_path     VARCHAR(500)     NOT NULL
  rows              INT              NOT NULL
  cols              INT              NOT NULL
  grid_data         LONGTEXT         NOT NULL  -- JSON array 2D: [[0,1,0,...],...]
  map_image_url     VARCHAR(500)     NULL
  is_active         TINYINT(1)       NOT NULL DEFAULT 1
  created_at        DATETIME         NOT NULL
  updated_at        DATETIME         NULL

  Ghi chú: grid_data lưu mảng 2D, 0=free cell, 1=obstacle
  grid_location = row * cols + col (tương thích MAPF format)
  Dùng bởi: #16 #17 #18 #19 #24 #31 #91

[T09] grid_pois (Point of Interest)
  poi_id            INT UNSIGNED     PK AUTO_INCREMENT

  poi_code          VARCHAR(30)      UNIQUE NOT NULL
  poi_name          VARCHAR(200)     NOT NULL FULLTEXT
  poi_type          ENUM('room','corridor','elevator','stairs',
                         'wc','entrance','pharmacy','canteen',

  grid_row          INT              NOT NULL

  grid_location     INT              NOT NULL INDEX  -- row * cols + col
  is_landmark       TINYINT(1)       NOT NULL DEFAULT 0
  is_accessible     TINYINT(1)       NOT NULL DEFAULT 1
  wheelchair_accessible  TINYINT(1)  NOT NULL DEFAULT 0
  custom_weight     FLOAT            NOT NULL DEFAULT 1.0
  capacity          SMALLINT         NULL
  details           JSON             NULL
  open_hours        VARCHAR(100)     NULL
  is_active         TINYINT(1)       NOT NULL DEFAULT 1

  Ghi chú: Mỗi POI = 1 ô cụ thể trên grid, được admin gán tên/loại
  grid_location phải trỏ đến ô free (grid_data[row][col] == 0)
  details JSON lưu thêm data (ssid, password, slots, menu...)
